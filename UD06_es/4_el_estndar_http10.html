4. El estándar HTTP/1.0

El estándar HTTP / 1.0 recoge, únicamente, tres órdenes que representan las operaciones de recepción, envío de la información y revisión del estado.

  • Método de petición GET . Se utiliza para solicitar un recurso del servidor. Siempre que pulsamos sobre un enlace o escribimos una dirección de Internet en la barra de navegación de un navegador web, estamos utilizando este método de petición. Como resultado, el servidor HTTP envía el documento correspondiente a la dirección de Internet seleccionada.
  • Método de petición HEAD . Solicita información sobre un recurso como, por ejemplo, su tamaño, el tipo, la fecha de modificación, etc. Suele ser utilizado por los gestores de memorias caché de páginas o por los servidores intermediarios (proxy server) para conocer cuando es necesario actualizar la copia que se mantiene de un archivo determinado.
  • Método de petición POST . Se utiliza para enviar información al servidor, como por ejemplo, los datos contenidos en un formulario. El servidor pasará esta información a un proceso encargado de su tratamiento (suele ser una aplicación del servidor). La operación que se llevará a cabo con la información proporcionada dependerá de la dirección de Internet (URL) utilizada, principalmente, en los formularios.

Cabe decir que un cliente web selecciona automáticamente las órdenes HTTP necesarias para recoger la información para el usuario. Por tanto, ante la activación de un enlace, siempre se ejecuta una operación GET con el fin de recoger el documento correspondiente.

Tal y como muestra la siguiente imagen, la comunicación entre el navegador y el servidor se lleva a cabo en dos etapas. Por un lado, el navegador hace una solicitud HTTP que, posteriormente, es procesada por el servidor que, en consecuencia, envía una respuesta HTTP.

Si evaluaremos con detenimiento la solicitud HTTP nos encontraríamos con un conjunto de líneas de texto que el navegador envía al servidor. Por un lado, encontraríamos una línea de solicitud , es decir, una línea que especifica el tipo de documento solicitado, el método que se aplicará y la versión del protocolo utilizado. Esta línea estará integrada por tres elementos separados por un espacio, es decir, el método, la dirección web y el protocolo HTTP utilizado por el cliente.

A continuación, encontraríamos los campos del encabezado de solicitud, es decir, un conjunto de líneas opcionales que permiten aportar información adicional sobre la solicitud y / o el cliente (navegador, sistema operativo, etc.). Cada una de estas líneas estará formada por un nombre que describe el tipo de encabezado, seguido por dos puntos (:) y por el valor del encabezado.

Finalmente, el cuerpo de la solicitud es un conjunto de líneas opcionales separadas de las líneas anteriores con una línea en blanco. Su finalidad es permitir que se envíen datos al servidor utilizando un formulario durante la transmisión.

Un ejemplo de solicitud HTTP sería el siguiente:

GET http://www.pego.org HTTP/1.0
Accept : Text/html If-Modified-Since : Saturday, 19-May-2006 17:39:13 GMT
User-Agent : Mozilla/6.0 (compatible; MSIE 5.0; Windows XP)

En consecuencia, la solicitud HTTP recibiría una respuesta HTTP, es decir, un conjunto de líneas que el servidor enviará al navegador web. Como en el caso de la solicitud, la respuesta HTTP contiene algunos elementos definitorios como, por ejemplo, una línea de estado. Esta línea especifica la versión del protocolo utilizada y el estado de la solicitud en proceso mediante un texto explicativo y un código.

En el caso de los campos de la cabecera de respuesta, este es un conjunto de líneas opcionales que permiten aportar información adicional sobre la respuesta y / o el servidor. Cada una de estas líneas se compone de un nombre que define el tipo de encabezado, seguido por dos puntos (:) y por el valor del encabezado.

En el caso del cuerpo de la respuesta, este contiene el documento solicitado. A modo de ejemplo, a continuación se muestra un ejemplo de una respuesta HTTP:

HTTP/1.0 200 OK Date: 19 May 2006 17:40:12 GMT Server : Microsoft-IIS/2.0
Content-Type : text/HTML Content-Length : 1245 Last-Modified : 11 May 2006 08:29:13 GMT

En cuanto a los códigos de respuesta , estos son los códigos que se ven cuando el navegador no puede mostrar la página web solicitada. La sintaxis de una respuesta HTTP es una línea de estado y tiene una estructura fija donde se indica la versión, el código del error y un texto explicativo.

Los posibles códigos de estado se identifican con números de tres cifras y se clasifican en cinco grupos según sean informativos (1xx), de éxito en la solicitud (2xx), para volver a dirigir la solicitud (3xx ), por un error generado en el cliente (4xx) o bien por errores generados en el servidor (5xx).

Las respuestas más típicas son los códigos 200 para indicar la confirmación de la petición y el 404 para indicar que el objeto solicitado no se encuentra disponible.

Línea de encabezado HTTP

Campo fundamental que define gran parte de la información que se intercambia entre los clientes y los servidores. Estas líneas permiten el envío de información descriptiva en la transacción y facilitan, por ejemplo, la autenticación o identificación de los usuarios.
