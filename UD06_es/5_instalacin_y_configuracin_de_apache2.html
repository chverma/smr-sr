5. Instalación y configuración de Apache2

La información la encontrará en https://help.ubuntu.com/lts/serverguide/httpd.html

directivas importantes

Directiva <Directory>.

La directiva Directory permite configurar permisos en los diferentes directorios de nuestro espacio web. Se trata de una directriz que se utiliza para definir características que sólo se aplicarán al directorio especificado y no a todo el servidor (por ello se ha de cerrar).

Estos permisos serían los siguientes:

  • Options
    • Indexes . Si dentro de la carpeta no existe ningún archivo igual que lo que hemos definido en el parámetro DirectoryIndex el navegador se mostrará un índice con todos los archivos de la carpeta actual.
    • FollowSymLinks . El servidor seguirá los enlaces simbólicos (lógicos), es decir, los alias.
  • Require  all denied, require not ip 192.168.1.50. Deniega la petición a una carpeta, un archivo o una dirección de Internet.
  • Require  all granted, require ip 192.168.1.50. Permite la petición a una carpeta, un archivo o una dirección de Internet.

Directiva <Files>

La directiva Filas permite configurar permisos en los diferentes archivos de nuestro espacio web. Cabe decir que el parámetro Files debe estar incluido dentro del parámetro Directory.

Estos permisos serían los siguientes:

  1. require all denied, require not ip 192.168.1.50. Deniega la petición a un archivo o una dirección de Internet.
  2. require all granted, require ip 192.168.1.50. Permite la petición a un archivo o una dirección de Internet.
  3. A modo de ejemplo, podríamos encontrar lo siguiente:
    1. Queremos prohibir el acceso a archivos con formato JPG dentro de la carpeta /var/www/serveisenxarxa/public_html.
<directory "/var/www/serveisenxarxa/public_html">
  <files *.jpg>
     require all denied
  </files>
  require all granted
</directory>

Directiva <VirtualHost>

La directiva VirtualHost permite configurar más de un sitio web en un mismo servidor, es decir, una misma dirección de Internet para diferentes nombres de dominio. Cabe decir que, si queremos utilizar este parámetro, es necesario haber configurado el servidor de nombres de dominio (DNS) para que dirija los nombres de dominio hacia nuestro servidor web Apache.
