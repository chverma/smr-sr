3. Gestión de usuarios y permisos

  • Usuarios locales: emplean las credenciales de los usuarios del sistema para acceder.
  • Usuarios genéricos: Se conecta con el usuario anonymous [Anónimo] y una clave libre (suele ser una cuenta de correo). Permite una conexión como invitado, con la que sólo se puede acceder a una parte del servicio.
  • Usuarios virtuales: No requiere una cuenta local del sistema. Se conecta y se autentica en una base de datos específica que no tiene conexión con el resto de servicios del sistema.

3.1. Acceso anónimo

Normalmente, si bien, el establecimiento de conexiones con servidores de archivos conlleva la necesidad de disponer de un nombre de usuario y una contraseña para autorizar tanto la conexión como el acceso a una cuenta dentro del servidor, dentro de Internet hay servidores de archivos que facilitan el acceso de forma anónima.

Los servidores de archivos anónimos suelen ser grandes "cajones de sastre" que contienen archivos de software -normalmente, de dominio público o de libre distribución-, archivos de imagen, de sonido y / o de vídeo, etc.

Por último, a pesar de que gran parte de los servidores de archivos anónimos suelen permitir, únicamente, la lectura de los archivos almacenados, también hay que facilitan la posibilidad de dejar archivos dentro de un directorio de entrada (incoming) que , de vez en cuando, es revisado por el administrador del servidor.

Dado que el protocolo FTP se desarrolló en el entorno de GNU / Linux, existe la posibilidad de otorgar diferentes permisos de ejecución, lectura y escritura a los archivos y, a la vez, establecer tres tipos de usuarios (Propietario, Grupo y otros). Normalmente, el Propietario es quien ha creado o ha subido el archivo al servidor. El Grupo se refiere a un grupo de usuarios al que puede pertenecer al propietario y, por último, los Otros son todos aquellos usuarios anónimos o que pertenecen al grupo indicado.

3.3. Cuotas

Un servidor de protocolo de transferencia de archivos (FTP, file transfer protocol) no es más que un programa que se puede instalar y ejecutar en cualquier ordenador conectado a cualquier tipo de red para facilitar el intercambio de datos entre el ordenador servidor y los ordenadores cliente. Ahora bien, la utilización de un ordenador conlleva que haya limitaciones de espacio.

En consecuencia, con el fin de respetar el espacio para el software presente en el ordenador y el de los archivos almacenados, dentro de las diferentes cuentas FTP existe un parámetro consistente en la cuota o el espacio en disco disponible. Por lo tanto, el tamaño de todo lo que haya almacenado dentro del servidor de archivos por parte de una cuenta FTP no puede superar la cuota establecida.

Si no definiéramos cuotas de espacio en un servidor de archivos, este quedaría saturado en un espacio de tiempo breve.

Es decir, podemos disponer de un servidor de archivos con varias cuentas definidos con diferentes cuotas de espacio cada una. Por ejemplo, si dispusiéramos de dos cuentas FTP diferentes, podríamos establecer una cuota de 100 MB y una cuota de 300 MB, respectivamente. En este caso, podríamos utilizar ambas cuentas siempre que el tamaño de los archivos almacenados en el servidor mediante una de las cuentas no excediera los 100 MB. En este caso, únicamente podríamos subir estos archivos mediante la cuenta con una cuota de espacio de 300 MB y almacenándolos en su espacio -con la condición de que no superáramos el espacio disponible de 300 Mb.

3.4. Límite de de ancho de banda

El límite del ancho de banda es la máxima cantidad de archivos que pueden ser transferidos entre el ordenador servidor y los navegadores web de los visitantes.

La mayoría de los sitios web disponen de una determinada cantidad de archivos que, en el momento de ser visitados mediante las diferentes páginas web, se descargan desde el servidor temporalmente en el ordenador mientras el usuario esté visitando la página web.

Normalmente, esta operación se realiza, por primera vez, cuando se visita la página web, ya que es entonces cuando los archivos quedan almacenados temporalmente en los directorios del navegador web. Es por este motivo que, en el caso de un sitio web con un gran número de visitantes, hay una transferencia masiva de los archivos temporales del sitio web y, en consecuencia, un alto consumo de ancho de banda. De hecho, esto puede conllevar que, en los momentos de mayor audiencia, el acceso al sitio web puede convertirse en imposible.

En el caso del protocolo de transferencia de archivos (FTP, file transfer protocol), la subida de archivos muy pesados ​​mediante un cliente FTP puede conllevar un alto consumo de ancho de banda y, en consecuencia, limitar el acceso al sitio web.
