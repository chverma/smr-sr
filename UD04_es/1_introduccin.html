1. Introducción

Si bien los usuarios de Internet disponen de herramientas como los blogs donde, a modo de bitácora, pueden insertar sus reflexiones, noticias u opiniones en línea, también tienen la posibilidad de agrupar un mayor volumen de información mediante una página web . De hecho, ante los diferentes formatos que puede tener la información a compartir (archivos de sonido, vídeo, documentos en PDF, etc.), debemos disponer de las herramientas para subir la información a la página web. En este caso, es el protocolo de transferencia de archivos (FTP, file transfer protocol) quien facilita estas tareas.

Un proceso de carga y descarga tanto se puede realizar estableciendo una sesión FTP desde el navegador, o bien, utilizando software especializado. Cabe decir que, muchas veces, por el tipo de archivos almacenados, una vez se ha establecido la conexión, se debe proporcionar una identificación y una contraseña asignada por el responsable del servidor de protocolo de transferencia de archivos (FTP, file transfer Protocol).

Una vez que se establece su identidad se abre un vínculo entre el ordenador cliente y el servidor. A partir de aquí las órdenes se envían y se ejecutan en el servidor y los resultados se devuelven al cliente. Esta comunicación permitirá navegar y manipular los diferentes archivos contenidos dentro del ordenador servidor.

Una vez se ha finalizado la transferencia de archivos, la conexión se terminará automáticamente. De hecho, después de finalizar la sesión, se puede producir la desconexión cerrando el vínculo de instrucciones y finalizando la sesión.

El Protocolo de transferencia de archivos (En inglés File Transfer Protocol, FTP) es un protocolo de red para la transferencia de archivos entre sistemas conectados a una red IP basado en la arquitectura cliente-servidor. Desde un equipo cliente se puede conectar a un servidor para descargar archivos desde él o para enviarle archivos, independientemente del sistema operativo utilizado en cada equipo.

Los objetivos de FTP son:

  1. Para promover el intercambio de archivos (programas y / o datos),
  2. Fomentar indirectamente el uso de ordenadores remotos.
  3. Hacer transparente al usuario de la variación en los sistemas de almacenamiento de archivos entre los hosts.
  4. Transferir datos de forma fiable y eficiente.

FTP, aunque se puede usar directamente por un usuario en un terminal, está diseñado principalmente para su uso mediante programas.

El protocolo de transferencia de archivos se suele utilizar para subir a Internet las páginas web una vez elaboradas y diseñadas para que puedan ser accesibles para el público desde Internet.
