2. Funcionamiento

Recuerda: puerto de red

Un puerto de red es una interfaz utilizada para comunicarse con un programa a través de una red. Suelen estar numerados y una determinada implementación de protocolo de transmisión de red (por ejemplo, TCP o UDP) asigna alguno de estos números a la información que envía; por ejemplo, el software de correo electrónico utiliza el puerto 25.

A diferencia de la mayoría de los protocolos utilizados en Internet, el servicio de protocolo de transferencia de archivos (FTP, File Transfer Protocol) necesita utilizar múltiples puertos de red para establecer las comunicaciones y las diferentes transferencias de datos.

Por defecto, los puertos que utiliza un servidor FTP son el 20 y el 21 donde, desde el punto de vista del servidor, el puerto 21 (también llamado puerto de órdenes) es el encargado de escuchar (o recibir) las órdenes provenientes del cliente así como de iniciar la conexión, mientras que el puerto 20 es el seleccionado para arrancar la conexión de datos con el cliente. Paralelamente, cualquier petición de datos desde el servidor hacia el cliente se realiza mediante el puerto 20, esto es el puerto de datos.

Llegados a este punto, cabe destacar que tanto el puerto elegido para la conexión de datos como la modalidad en que se inician las conexiones dependen de la opción de que el cliente seleccione para la transferencia de datos desde el servidor de archivos, esto es, una transferencia en modo activo o pasivo.

Cuando establecemos una comunicación con el servidor de archivos en modo activo establecen dos conexiones diferentes donde el servidor establece la conexión desde el puerto 20 a un puerto aleatorio sin privilegios especificado por el ordenador que actúa como cliente.

En el modo pasivo, es el ordenador cliente quien inicia la conexión dirigiéndose al servidor de archivos para solicitar el acceso a los datos y la apertura de los puertos correspondientes para el establecimiento de las conexiones de control y datos. En consecuencia, los cortafuegos presentes en los ordenadores cliente no detectan ningún tipo de conexión desde el exterior que pueda ser considerada peligrosa.

Cortafuegos

Dispositivo configurado para controlar conexiones de datos según una política de seguridad. La tarea básica de un cortafuegos (firewalls) -en forma de hardware o software- es controlar el tráfico entre redes estableciendo diferentes niveles de confianza como, por ejemplo, entre la red de Internet y una red de área local.
