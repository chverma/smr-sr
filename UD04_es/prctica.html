Práctica

Actividad 4.1: Instalación y Configuración FTP anónimo

Objetivos

  • Aprender a instalar y configurar un servicio FTP en GNU / Linux.
  • Entender el concepto de usuario virtual.
  • Trabajar con los permisos en el sistema de archivos.

Actividades

  1. Instala y configura el servidor FTP (vsftpd) para que permita el acceso anónimo para sólo lectura a una carpeta llamada público.
  2. Configurar el DNS para que tenga nombre el servidor ftp. (ftp.midominio.com).
  3. Agregar algunos archivos a esa carpeta y comprueba el funcionamiento.
  4. Permite al usuario anónimo colgar archivos en una nueva carpeta llamada incoming.
  5. Asegúrate de que los archivos se pueden descargar tras colgarlos.

Recursos

  • Instalación de un servidor FTP en Ubuntu Server Guide: https://help.ubuntu.com/lts/serverguide/ftp-server.html
  • Información de las opciones de configuración: https://security.appspot.com/vsftpd/vsftpd_conf.html 
  • Las opciones que afectan a los usuarios anónimos empiezan por anon_
  • Ten en cuenta que el usuario virtual necesita disponer de un usuario local, en este caso es ftp.

Entregable

  1. Manual de pasos realizados para la instalación y para la configuración.
  2. Capturas de pantalla que muestran el funcionamiento. Sólo lectura, acceso al directorio, permisos, etc.

Actividad 4.2: FTP en modo consola

Prueba los siguientes comandos del cliente FTP en modo consola:

  • help
  • ascii
  • binary
  • cd
  • chmod
  • close
  • quit
  • bye
  • delete
  • mkdir
  • get
  • dir
  • ls
  • put
  • pwd
  • prompt
  • rename
  • rmdir
  • size
  • passive
  • open

Hay también una serie de pedidos interesantes que empiezan por "m" (ej: mput, mget, ...)

Después de haber probado como funciona el cliente FTP en modo consola elabora una tabla resumen de los pedidos que has utilizado.

Actividad 4.3: Configuración VSFTP para usuarios autenticados

Manual de vsftpd.conf

Recuerda: Permisos en Linux

Configurar el servidor FTP (vsftpd)

  1. Modifica la tasa de transferencia para los usuarios anónimos a 10 bytes por segundo. ( anon_max_rate)
  2. Gestionar conexiones máximas por IP (2) [ max_per_ip] y máximo número de clientes (5) [ max_clients].
  3. Ajusta los "timeouts" [ timeout...]
  4. Ajusta el servidor para permitir acceso a algunos de los usuarios del sistema (no a todos) [ local_enable, userlist_file,  userlist_enable]. Crear un nuevo usuario ftp_user que será el único que tendrá acceso. [ adduser]
  5. Documento el proceso, explicando los cambios realizados.
