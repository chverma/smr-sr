5. Tipos de registros

El sistema de nombres de dominio es una base de datos jerárquica y distribuida en el que cada servidor de nombres gestiona la información correspondiente a la zona de la que es autoritario. Cada zona contiene información de los hosts que la forman. La información de zona se almacena en forma de registro de recurso o resource recuerdo (RR). Existe la información que permite identificar cada nombre de dominio con la dirección IP correspondiente, llamado forward mapping o resolución directa. También contiene la información para identificar cada dirección IP con el nombre de dominio correspondiente, llamado reverse mapping o resolución inversa. La información de zonas contiene otras informaciones que permiten identificar los servidores DNS autoritarios por la zona, los servidores de correo, etc.

5.1. Base de datos de zona

La configuración de una zona se almacena en un conjunto de archivos llamado ficheros de zona. La especificación del DNS dice cómo deben ser estos archivos de zona y cómo se describirán los registros de recurso (descripción de cada elemento que pertenece a la zona).

El conjunto de los registros de recurso de todas las zonas del espacio de nombres forman la base de datos distribuida y jerárquica del sistema DNS.

En cualquier zona habrá menos los archivos de zona siguientes:

  • Un archivo con las asociaciones de los nombres de dominio a direcciones IP. Este archivo define la resolución directa.
  • Un archivo para cada subred con la asociación de cada dirección IP a su nombre de dominio canónico. Define la resolución inversa.
  • Un archivo con la definición de la resolución inversa del loopback.
  • Un fichero con la descripción de los nodos raíz de Internet.

Una vez los archivos de zona contienen todos los registros de recurso necesarios hay que configurar el servidor de nombres para que utilice estos archivos. Si bien la configuración de los ficheros de zona es estándar (definida por la especificación DNS), la configuración del servidor depende del programa que se utilice.













La base de dades en bind9 (Ubuntu Server)








Los siguientes archivos son básicos para la configuración de una zona para el dominio example.org:






;/etc/bind/named.conf.local
zone "example.org" {
	type master;
	file "/etc/bind/db.example.org";
};







En este caso ns.example.org será nuestro servidor de nombres autoritario:






; /etc/bind/db.example.org
;
; BIND data file for example.org
;
$TTL    604800
@       IN      SOA     ns.example.org. hostmaster.example.org. (
                             2         ; Serial
                        604800         ; Refresh
                         86400         ; Retry
                       2419200         ; Expire
                        604800 )       ; Negative Cache TTL
       IN      A       192.168.1.10
;
@       IN      NS      ns.example.org.
@       IN      A       192.168.1.10
ns      IN      A       192.168.1.10






Esta sería la configuración mínima, ahora habría que ir añadiendo entradas en el registro.
