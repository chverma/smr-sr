4.3. Resolución inversa

resolución inversa

El DNS proporciona un mecanismo para obtener el nombre de dominio al que corresponde una dirección IP. Este mecanismo, llamado resolución inversa, se basa en un dominio especial llamado in-addr.arpa. Hay protocolos de red que requieren una resolución inversa correcta para funcionar bien ya menudo se utiliza como medida de seguridad para verificar la existencia de la dirección IP en un dominio.

Se ha ideado un dominio de nombre in-addr.arpa que permite representar en forma de nombre de dominio todas las direcciones IP posibles. El formato son etiquetas numéricas del 0-255 que representan cada octeto de una dirección IP. Las etiquetas de los bytes se concatenan en orden inverso y se les añade el sufijo in-addr.arpa. Un nombre de dominio con cuatro etiquetas de bytes corresponde a un nodo, un nombre de dominio con menos etiquetas corresponde a una red.

En el ejemplo siguiente se muestran los servidores de nombres del dominio ioc.cat y se hace una consulta de resolución inversa a cada uno de ellos:

[root@localhost ~]# host -vt NS ioc.cat
Trying "ioc.cat"
;; ->>HEADER<<- opcode: QUERY, status: NOERROR, id: 6116
;; flags: qr rd ra; QUERY: 1, ANSWER: 2, AUTHORITY: 0, ADDITIONAL: 2

;; QUESTION SECTION:
;ioc.cat. IN NS

;; ANSWER SECTION:
ioc.cat. 900 IN NS dns1.nominalia.com.
ioc.cat. 900 IN NS dns2.nominalia.com.

;; ADDITIONAL SECTION:
dns1.nominalia.com. 145929 IN A 62.97.103.68
dns2.nominalia.com. 145925 IN A 213.151.118.169

Received 108 bytes from 80.58.61.250#53 in 112 ms

[root@localhost ~]# host 62.97.103.68
68.103.97.62.in-addr.arpa domain name pointer dns1.nominalia.com.

[root@localhost ~]# host 213.151.118.169
169.118.151.213.in-addr.arpa domain name pointer dns2.nominalia.com.
