5.3. Registro NS

El registro de recurso NS o name server (servidor de nombres) define un servidor de nombres autoritativo para la zona. Habrá tantas entradas NS como servidores de nombres autoritativos hay en la zona. El estándar DNS en recomienda al menos dos (uno primario o master y uno de seguridad secundario o slave).

Un registro NS consta de los campos:

nombreDominio. IN NS nameServer.

Un ejemplo sería este:

inf.ioc.cat. IN NS ns1.inf.ioc.cat.

La descripción de cada campo es la siguiente:

  • nombreDominio. indica el nombre del dominio que se está definiendo.
  • IN indica que la clase es Internet.
  • NS describe que se trata de un tipo de registro de recurso en el que se define un servidor de nombres.
  • nameServer. es el nombre del servidor de nombres. Fíjese de nuevo que tanto nombreDominio. como nameServer. terminan en un punto para indicar que son nombres de dominio absolutos o FQDN.
