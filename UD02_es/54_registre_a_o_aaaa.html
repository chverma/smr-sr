5.4. Registre A o AAAA

Un registro de recurso A o address (dirección) asocia un nombre de host a una dirección IP (resolución directa). Por cada nombre de host de la red deberá disponer de una entrada donde se asocie el nombre del host a su dirección IP.

Un registro A consta de los campos:

nombreHost. IN A IP

Un ejemplo sería este:

mahatma.inf.ioc.cat. IN A 192.168.0.2

La descripción de cada campo es la siguiente:

  • nomHost. indica el nombre del host que se está definiendo.
  • IN indica que la clase es Internet.
  • A describe que se trata de un tipo de registro de recurso de definición de dirección IP.
  • IP es la dirección IP asignada al host.

Fijémonos de nuevo que nomHost. acaba en punto para indicar su FQDN. Un host puede tener más de una IP asignada al mismo nombre de host y se denomina multi-homed. Simplemente será necesario que haya un registro A para cada dirección IP. Constará del mismo nombre de host a la izquierda de la definición, y la correspondiente dirección IP a la derecha. Por ejemplo:

superserver.dom.com. IN A 10.0.0.1
superserver.dom.com. IN A 10.0.0.2

Los nombres definidos en los registros de tipo A son nombres canónicos. Un host se puede identificar por más de un nombre, pero sólo uno es el nombre canónico (original), el resto son alias. Los nombres canónicos se definen con el tipo de registro A. Los alias se definen con el tipo de registro CNAME.

Actividad: Registros de Recursos

A través de las de consulta DNS, dig, host o nslookup obte la siguiente información:

  1. El RR de tipo SOA para la zona wikipedia.org. Necesitarás marcar la opción Dig, dejar vacío el campo siguiente e incluir la zona en el campo Host / IP.
  2. El RR de tipo NS para la zona wikipedia.org
  3. El RR de tipo A para la zona wikipedia.org
  4. El RR de tipo CNAME para la zona www.wikipedia.org

De cada registro deberás indicar los datos comunes (propietario, TTL, clase, tipo) así como los datos propios de cada uno, explicando que significa cada campo.
