6. Evolución del protocolo y seguridad

Los protocolos DNS han evolucionado mucho gracias a las nuevas necesidades provocadas por el incremento exponencial de las redes. Una de las principales vías en evolución es el DDNS o dinámico DNS y el DNSSEC o DNS seguro.

DDNS

El protocolo DDNS (dynamic DNS) permite que los datos del servidor DNS se actualicen en tiempo real. El principal uso es permitir que clientes con direcciones IP dinámicas de intervalo puedan disponer de un nombre de dominio (a pesar de que su dirección IP varía de una sesión a otra). Un mecanismo consiste en permitir que los servidores DHCP se comuniquen con los servidores DNS y los notifiquen las actualizaciones en la base de datos de DNS que hay que hacer. En función de las configuraciones de red que los servidores DHCP proporcionan a sus clientes se hacen cambios en el DNS.

DNSSEC

Hombre en el medio

Se llama man-in-the-middle aquellos equipos que se hacen pasar por otros en una conexión de red. El cliente cree conectar con su banco, pero de hecho está conectando con un"adelante"situado en medio. Este atacante recibe el tráfico del cliente y lo transfiere al banco, recibe la respuesta del banco y la pasa al cliente. En este proceso está en disposición de"manipular"todo este tráfico a su conveniencia.

DNSSEC o Domain Name System Security Extensions (extensiones de seguridad para DNS) son un conjunto de especificaciones de seguridad para permitir una comunicación DNS segura, de manera que el cliente pueda estar plenamente seguro de que quien le responde las consultas es su servidor DNS y no un impostar (man-in-the-middle). También garantiza la integridad de los datos tanto de las consultas como de las respuestas y además está diseñado para prevenir ataques de denegación de servicio.

Más información: https://www.icann.org/resources/pages/dnssec-qaa-2014-01-29-es

Servidores DNS envenenados

Uno de los principales problemas del protocolo DNS (como de todos los primeros protocolos de Internet) es la falta de seguridad. Fue diseñado en una época de 'buen rollo' donde se confiaba con los otros integrantes de la red. Esto hoy en día no es demasiado sensato.

Imagínese que un atacante consigue el control de un servidor DNS o consigue hacer pasar su servidor DNS falso como servidor de un conjunto de usuarios. Cada vez que estos clientes hacen una consulta en Internet, por ejemplo en su banco (poniendo el nombre de la web), el servidor DNS envenenado proporciona una dirección IP no del banco real sino de una web falsa con el fin de ...

Cada vez que accedemos a una web, un servicio, un host, etc. por medio de su nombre de dominio, confiamos en que el servidor DNS proporcione la dirección IP correcta. Somos confiados, eh?
