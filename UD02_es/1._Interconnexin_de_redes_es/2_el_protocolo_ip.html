2. El protocolo IP

La versió més usada actualment del protocol IP és la IPv4, definida en l’RFC 791 del 1981. Actualment ja disposa d’un successor, la IPv6, l’ús de la qual es va estenent progressivament.

Totes les versions del protocol IP permeten l’enviament de paquets entre equips sense establir cap mena de connexió. Això vol dir que l’ordinador amfitrió origen envia dades al destinatari sense esperar cap mena de notificació que les dades s’han rebut correctament.

Avui dia, el TCP/IP és el conjunt de protocols escollit per a la immensa majoria de les xarxes actuals i, per descomptat, per a qualsevol sistema que es vulgui connectar a Internet.

Tot i que per enviar dades entre dos ordinadors amfitrions ja n’hi ha prou amb el protocol IP, no ofereix cap mena de garantia que s’enviïn correctament les dades o, ni tan sols, que arribin a la destinació. De fet, no ens garanteix que les dades, si arriben a la destinació, estiguin intactes, ja que el control d’errors només es fa sobre les capçaleres, no sobre les dades que transmet. Així doncs, tot i que es pot fer servir IP directament, per a aplicacions que requereixen fiabilitat fan servir al protocol de la capa de transport TCP.

Todas las versiones del protocolo IP permiten el envío de paquetes entre equipos sin establecer ningún tipo de conexión. Esto significa que el equipo host origen envía datos al destinatario sin esperar ningún tipo de notificación que los datos se han recibido correctamente.

Aunque para enviar datos entre dos ordenadores anfitriones ya basta con el protocolo IP, no ofrece ningún tipo de garantía que se envíen correctamente los datos o, ni siquiera, que lleguen al destino. De hecho, no nos garantiza que los datos, si llegan al destino, estén intactos, ya que el control de errores sólo se hace sobre las cabeceras, no sobre los datos que transmite. Así pues, aunque se puede usar IP directamente, para aplicaciones que requieren fiabilidad utilizan el protocolo de la capa de transporte TCP

2.1. Configuración de un nodo IP

En primer lugar, habrá que configurar los protocolos TCP / IP locales que suelen incluir en el núcleo de cualquier sistema operativo moderno.

A la hora de configurar el equipo, habrá que dispondrán er de la siguiente información:

  • El dirección IP, ya sea de la versión 4 o de la nueva versión 6.
  • La máscara de subred, que sirve para identificar la red o subred.

    Si la comunicación se establece con equipos de otras subredes o con Internet, se necesitará además:

    • La dirección IP de la puerta de enlace (gateway ) o pasarela por omisión que corresponde al router y con el que se redirige el tráfico en Internet o a otras subredes.
    • Las direcciones IP de los servidores DNS.

El protocolo IP tiene las siguientes funciones: </ p>

  • Elegir el camino más adecuado para enviar los paquetes.
  • Implementa un servicio de envío con las características siguientes:
    • No orientado a la conexión. Cada paquete puede seguir un camino distinto. Por lo que pueden llegar desordenados.
    • No fiable. Los paquetes pueden perderse, dañarse o llegar con retraso.
    • Proporciona un mecanismo de direccionamiento lógico (direcciones IP).

Direccionamiento IP

La dirección IP es un número de 32 bits que identifica cada una de las máquinas que están conectadas a Internet o en cualquier red, así como la red a la que están conectadas. Una parte de la dirección IP, a su máscara de red, sirve para identificar la red, siendo el trozo restante de la dirección IP la que identifica la máquina.

Para poder separar el campo que identifica la red del campo que identifica la máquina, se debe aplicar una máscara de red. Es decir, al aplicar la operación lógica AND entre la máscara de red y la dirección IP se obtiene la red.

Clases de dirección IPv4

Hay tres clases principales de direcciones IP:

  • Dirección de clase A: el primer octeto identifica la red.
  • Dirección de clase B: los dos primeros octetos identifican la red.
  • Dirección de clase C: los tres primeros octetos identifican la red.

    Además de dos clases adicionales que se tratan por separado:

    • Dirección de clase D: Se trata de un conjunto de direcciones reservadas para multidifusión.
    • Dirección de clase E: Se trata de una clase reservada.

Direccionamiento sin clase (CIDR)

Con la rápida expansión de Internet quedó claro que el direccionamiento basado en clases no era suficiente, por lo que en 1993 se propuso la CIDR (classless inter-domain routing) que significa encaminamiento entre dominios sin clase. Gracias a este sistema se mejora el método con el que se interpretan las direcciones IP además de cómo se encaminan los paquetes.

En lugar de hablar de clases se utiliza el número de bits a 1 de la máscara de red para indicar la red. Las clases A, B y C tendrían una máscara de red con 8, 16 y 24 bits a 1 respectivamente. Por ejemplo, para definir la red que va del 192.168.0.0 a la 192168255255 (sería una clase B) indicamos la IP y con un barra los 16 bits a 1 de la máscara de red. Por lo tanto la notación sería: 192.168.0.0/16

Para poder hacer uso de CIDR, los routers de la red deben ser capaces de interpretar direcciones IP que no pertenecen a ninguna de las clases convencionales (A, B o C).

Direcciones reservadas y/o especiales

  • Quan l’identificador d’equip és 0, es fa referència a la xarxa a la qual està connectat.
  • Quan l’identificador d’equip són tot 1 vol dir totes les màquines; això seria una adreça de difusió (broadcast).
  • Quan tota l’adreça són 0 indica totes les IPs de la màquina.
  • Adreça de loopback. La xarxa 127.0.0.0/8 indica que el paquet es que a la mateixa màquina i retorna internament, es refereix sempre al equip local.
  • Adreces privades. Són adreces que només es poden utilitzar dins d’una organització privada no encaminables. Els blocs d’adreces privades són els següents:
    • 10.0.0.0/8: és una classe A que permet 224 hosts a la xarxa. Per la grandària que té se sol dividir en subxarxes.
    • 172.16.0.0/12: és una xarxa que admet fins a 220 equips connectats a la xarxa
    • 192.168.0.0/16: és una classe C que permet fins a 216 equips. Encara que sigui la més petita de les tres és la més comuna.
  • Cuando el identificador de equipo es 0, se hace referencia a la red a la que está conectado. Por ejemplo: 10.0.0.0/8
  • Cuando el identificador de equipo son todo 1 significa todas las máquinas; esto sería una dirección de difusión (broadcast). Por ejemplo: 10.255.255.255/8
  • Cuando toda la dirección son 0 indica todas las IPs de la máquina. Por ejemplo: 0.0.0.0/0
  • Dirección de loopback. La red 127.0.0.0/8 indica que el paquete es que en la misma máquina y devuelve internamente, se refiere siempre al equipo local.
  • Direcciones de enlace local: El objetivo de la red 169.254.0.0/16 es proveer de una dirección IP a los nodos sin necesidad de tener disponible un servidor DCHP. Estas direcciones no son encaminables.
  • Direcciones privadas. Son direcciones que sólo se pueden utilizar dentro de una organización privada no encaminables. Los bloques de direcciones privadas son los siguientes:
    • 10.0.0.0/8: es una clase A que permite 224 224 hosts en la red. Por el tamaño que tiene se suele dividir en subredes. 
    • 172.16.0.0/12: es una red que admite hasta 220 equipos connectados a la red.
    • 192.168.0.0/16: es una clase C que permite hasta 254 equips. Aunque sea la más pequeña de las tres es la más común.

Métodos de transmisión

Las métodos de transmisión pueden ser los siguientes.

  • Unidifusión ( Unicast ). El destinatario es un dispuesta.
  • Multidifusión ( Multicast ). Los destinatarios son un grupo de dipositivos.
  • Difusión ( Broadcast). Los destinatarios son todos los dispositivos de la red.

Configurar targeta de xarxa usant el terminal

1. Identificar la interfaz de red

En los sistemas basados ​​en Linux las interfaces de red (NIC) tienen un nombre lógico. Históricamente se emplea la nomenclatura eth0, eth1 o wlan01, etc. Actualmente los nuevos sistemas identifican las interfaces según la posición que ocupan en el bus del sistema ( Predictable Interface Names ). Así es más sencillo identificarlas.

Usaremos ifconfig -a para listar todas las interfaces de red.

2. Configurar la interfaz con asignación de dirección dinámica IP (Cliente DHCP)

Para configurar una interfaz debe editar el archivo /etc/network/interfaces con un editor de textos como nano.En el ejemplo asumimos que queremos configurar la interfaz enp2s0.

 auto enp2s0
iface enp2s0 inet dhcp

Añadiendo la configuración como se muestra más arriba puedes activar manualmente mediante el comando ifup que inicializa el proceso DHCP.

 sudo ifup enp2s0

Para desactivar la interfaz, puedes usar la orden ifdown </ code>, esto iniciará la desactivación del proceso DHCP y apagará la interfaz.

 sudo ifdown en2s0
