5.2. Registrar SOA

El registro de recurso SOA o start of authority (inicio de definición de zona con autoridad) dice que el archivo de zona donde se encuentra es la mejor fuente de datos para la zona, que el servidor de nombres es autoritario para la zona. Suele ser el primer RR que hay en el archivo de zona, aunque no es obligatorio. Por cada archivo de zona debe haber sólo un registro SOA.

Un registro SOA tiene el formato:

nombreDominio. IN SOA nsPrimario. admin.nsPrimario. (opciones-slaves)

Un ejemplo sería el siguiente:

inf.ioc.cat. IN SOA ns1.inf.ioc.cat. admin.ns1.inf.ioc.cat. ( 1h 3h 1h 1w 1h )

Punto final en el nombre de dominio

Poner, o no, el punto al final de un nombre de dominio es importante. Si termina con punto es un nombre de dominio absoluto. Si termina sin punto, es un nombre relativo y se añadirá el dominio por defecto al final.

La descripción de cada campo es la siguiente:

  • nombreDominio. indica el nombre del dominio que se está definiendo y por el que el servidor de nombres es autoritario. Fíjese en el punto al final del nombre del dominio, es importante ponerlo.
  • IN indica que la clase es Internet.
  • SOA describe que es un registro de recurso tipo SOA.
  • nsPrimario. es el nombre del host servidor de nombres primario para esta zona. Otra vez lea sobre el punto del final.
  • admin.nsPrimario. es la dirección de correo electrónico del administrador del servidor de nombres de dominio, con el formato usuario.servidor. El primer punto que separa el nombre de usuario y el nombre del servidor hay que interpretarlo como si fuera una @ (usuario @ servidor).
  • opciones-slaves son parámetros que indican entre paréntesis y que sirven para definir cómo debe ser la comunicación entre el servidor primario (o Maestro) Y los servidores secundarios (o esclavos). A grandes rasgos se indican los siguientes conceptos:
    • De serie: El número de serie de la versión de los datos. En cada cambio de los datos de la zona, el número se incrementa.
    • Actualizar: Tiempo a transcurrir entre cada refresco de datos del servidor secundario.
    • Rever: Tiempo de espera para volver a intentar un refresco si el servidor secundario ha fallado en el intento de actualizar sus datos del servidor primario.
    • Expire: Tiempo a partir del cual los datos del servidor secundario se consideran sin autoridad si no se han refrescado antes.
    • Mínimo: Valor del TTL de los campos por defecto. Recuerde que cada campo se puede asignar un TTL específico. Según la versión del servidor indicará el TTL de las respuestas negativas (negative caching), es decir, el tiempo que una respuesta negativa se almacenará encaché.
