4. Resolución de nombres

4. Resolución de nombres

A menudo en las aplicaciones de usuario y de sistema se accede a recursos por su nombre de dominio. Por ejemplo, un cliente web requiere una determinada página web, un navegador de ficheros quiere acceder a unas carpetas de una máquina remota que se identifican por el nombre de dominio, el sistema debe validar al usuario contra un servidor LDAP remoto, etc . En cada uno de estos casos habrá que resolver una pregunta del tipo este dominio a qué dirección IP corresponde? Esta pregunta no la responden las aplicaciones individualmente (el navegador web, el cliente de autenticación ...), sino que utilizan el Resolver para hacerlo.

El Resolver es la parte cliente de la arquitectura cliente-servidor del DNS. Él ha de atender las necesidades de las aplicaciones, confeccionar una consulta o query, hacerla a un servidor DNS, obtener la respuesta y pasarla a la aplicación pertinente.

La resolución

El mecanismo de resolución de nombres DNS consta de un cliente resolverlo que realizará las consultas (o querys) a resolver a unos servidores DNS.

Si el servidor dispone de la información porque forma parte de la base de datos de su zona, emitirá una respuesta autoritativa. Si dispone de la respuesta porque la tiene almacenada temporalmente (en un proceso llamado cache) también emitirá la respuesta pero esta vez de manera no autoritativa. Si no tiene información del dominio buscado, el servidor puede hacer a otros servidores la misma consulta en un proceso que puede ser recursivo o iterativo. Siempre existe un camino para encontrar el dominio buscado, que es preguntar a los nodos raíz (root servers) del espacio de nombres de dominio. Partiendo de los nodos raíz y recorriendo el árbol hacia abajo, se puede llegar al dominio buscado, si es que existe.

Siempre hay un camino a un dominio existente partiendo del nodo raíz. Cuando un servidor es consultado sobre un dominio que desconoce (no es de su zona ni tiene la respuesta en la cache) puede escalar la pregunta a un servidor de la raíz (root name server). Esto significa que los servidores raíz son cruciales para el funcionamiento del DNS.

Recursión e iteración

Cuando el cliente resolverlo emite una consulta al servidor DNS local (el servidor de nombres que tiene configurado), éste la puede tratar de manera recursiva o iterativa. De hecho, el cliente Resolver ya hará la consulta indicando si exige una respuesta recursiva o iterativa. La diferencia entre un modo y otro es como debe actuar el servidor DNS para obtener la respuesta cuando no la tiene en su base de datos de información.

En modo iterativo el servidor devuelve la mejor respuesta posible basada en su información local, sin preguntar a nadie más. En el modo recursivo el servidor intenta encontrar la respuesta preguntando a tantos otros servidores como sea necesario con el fin de obtenerla.

Un servidor puede emitir las siguientes respuestas:

  1. Responde enviando el dato que le han solicitado (un nombre de host, una dirección IP, la lista de servidores de nombres, de servidores de autenticación ...).
  2. Ha localizado el dominio buscado, pero no se dispone del dato solicitada. Hay que tener en cuenta que se pueden solicitar otros datos aparte de la dirección IP del dominio (por ejemplo, qué servidores de correo tiene el dominio).
  3. Finalmente puede que el dominio solicitado no exista.
