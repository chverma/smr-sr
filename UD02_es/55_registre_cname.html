5.5. Registre CNAME

Los registros de recurso CNAME o canonical name (nombre canónico) asocian un alias a un nombre canónico.

Un registro CNAME consta de los campos:

nomHost. EN CNAME hostCanonicalName. | IP

Un ejemplo sería este:

ftp.inf.ioc.cat. IN CNAME mahatma.inf.ioc.cat.
tftp.inf.ioc.cat IN CNAME 192.168.0.2

La descripción de cada campo es la siguiente:

  • nomHost. indica el nombre del alias que se está definiendo.
  • IN indica que la clase es Internet.
  • CNAME describe que se trata de un registro de recurso de definición de un alias.
  • hostCanonicalName | IP es el nombre de host canónico al que se asigna el alias.

Fíjese de nuevo que es un FQDN y termina en punto. Generalmente, los registros CNAME tienen en la parte derecha de la definición un nombre canónico, pero a veces será necesario indicar una dirección IP. Considere Unhost multi-homed con múltiples direcciones IP que además tiene alias. Si la definición fuera por el nombre canónico del host, no se sabría cuál de las direcciones IP corresponde al alias. En estos casos, el CNAME apunta a una dirección IP.

Ejemplo de un host multi-homed

Se quiere poner el alias super1 y super2 cada una de las IP del host superserver.com (un host que tiene dos direcciones IP asignadas a este nombre). Las entradas CNAME serían las siguientes:

super1.dom.com. IN CNAME 10.0.0.1.
super2.dom.com. IN CNAME 10.0.0.2.

La resolución de los alias se obtiene buscando la entrada del alias en el archivo de zona. Con la entrada CNAME obtiene el nombre canónico correspondiente al alias. De nuevo se vuelve a buscar en el archivo de zona, ahora el nombre canónico. Una entrada de tipo A proporcionará la dirección IP correspondiente (alias ->CNAME ->nombre canónico ->A –>dirección IP). Un alias nunca puede aparecer en la parte derecha de una definición de registro de recurso.
