1. Introducción

El sistema de nombres de dominio DNS (Domain Name System) proporciona un mecanismo eficaz para hacer la resolución de nombres de dominio a direcciones IP. Como usuarios (humanos) nos es más fácil dirigirnos a un nombre de dominio (de host, de web, de servidor de correo, etc.) utilizando un texto identificativo (por ejemplo, www.iesevalorpego.org) que a la dirección IP pertinente (por ejemplo, 213.73.40.230). El servicio DNS no sólo permite hacer la resolución de nombres de dominio a direcciones IP, sino también la resolución inversa. Es decir, a partir de una IP averiguar el nombre de dominio.

El servicio DNS proporciona independencia del nombre de dominio respecto a la IP. Así un dominio puede cambiar de IP de forma transparente para los usuarios del dominio. Incluso es usual que un dominio se identifique con más de una IP como medida de redundancia contra la caída del sistema o como balanceo de cargas. Otros servicios proporcionados por el DNS son la identificación de los servidores de correo de un dominio, de cada uno de los hosts que pertenecen a la red, servidores de impresión, etc.

ARPANET

ARPANET es la red de conmutación de paquetes que desarrolló el Departamento de Defensa de Estados Unidos y que fue la predecesora de Internet.

Un sistema de nombres plan se basa en la utilización de un archivo de texto que describe cada host con su correspondiente dirección IP. Se puede usar para definir alias para equipos locales en redes pequeñas, pero no es escalable a redes grandes y mucho menos en Internet.
