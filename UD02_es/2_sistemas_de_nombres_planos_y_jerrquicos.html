2. Sistemas de nombres planos y jerárquicos

El problema de identificar los equipos se produce desde el principio de la existencia de las redes de ordenadores y no es específico de las redes TCP / IP. Hay un mecanismo en"lenguaje humano"para identificar los equipos de la red. En especial los que proporcionan servicios a los demás equipos y usuarios. En la red inicial ARPANET, los equipos ya recibían un nombre. Estos nombres se hacían públicos por medio de un archivo centralizado que contenía los nombres de todos los equipos de la red y su identificación. Este archivo era el archivo hosts.txt (en sistemas GNU / Linux, conocida como / etc / hosts).

Ejemplo de archivo de nombres plan para describir los alias de una red local en el hogar:

[root@portatil ~]# cat /etc/hosts
# Do not remove the following line, or various programs
# that require network functionality will fail.
127.0.0.1 localhost.localdomain localhost localhost
::1 localhost6.localdomain6 localhost6
192.168.1.1 router routerWF
192.168.1.31 server1 escritorio padre
192.168.1.32 estacion1 dormitorio madre
192.168.1.33 estacion2 nino juegos supercrack

Poner nombres a los ordenadores de casa:

El archivo anterior pertenece al PC llamado portátil y pone nombres"caseros"los otros ordenadores de la familia. Así por ejemplo el equipo 192.168.1.33 se puede llamar de las cuatro maneras diferentes que se indican.

En una red pequeña se puede generar un archivo con el nombre e identificador IP de todos los hosts, centralizado en un servidor, y encargarse de distribuir copias de este archivo a todos los equipos de la red. Pero este modelo de conocimiento no es escalable. Si la red crece es imposible de mantener. Utilizar este modelo significaría que hay un equipo que centraliza los nombres de todos los hosts de Internet en un solo archivo! Por otra parte, también significaría que este archivo se repartirá entre todos los equipos de Internet para que sepan cómo se llaman los otros equipos cada vez que hay una actualización. Evidentemente hay otra solución.

Problemas de actualización de una red

Imagínese los problemas que se presentan en querer hacer llegar un mismo archivo a todos los hosts. Habrá inconsistencias con equipos que han recibido la actualización y otras que no. El tráfico de copiar el archivo en cada equipo. Un nuevo equipo en Internet significa escribir de nuevo el archivo y volverlo enviar a todo el mundo!

En 1983 surge el Domain Name System (DNS) para aportar una solución escalable y práctica. El DNS se basa en una base de datos de nombres de dominio jerárquica y distribuida:

  • Jerárquica porque cada nodo depende de un elemento padre (dominio superior), excepto el nodo raíz. Es decir se organiza en una estructura de dominios que se pueden componer de subdominios que también se pueden dividir en subdominios y así hasta 127 niveles (originariamente). Estos dominios son gestionados por servidores DNS responsables de cadazona.
  • Distribuida porque la información no está toda junta en un solo repositorio central, sino que la información se encuentra repartida por partes en los servidores DNS de Internet. Cada servidor DNS autoritario contiene la base de datos de suzona.

Reflexiona

Árboles de los sistemas DNS y de ficheros

El sistema de nombres de dominio tiene una estructura similar al árbol que se utiliza para representar un sistema de ficheros. La diferencia radica en el hecho de que las rutas de los sistemas de archivos indican desde la raíz hasta el nodo (por ejemplo, /etc/bind/bind.conf), y en cambio en el DNS las rutas indican del nodo inferior a la raíz (ns1.serviciosenred.org).

2.1. Elementos del espacio de nombres de dominio

El servicio DNS permite identificar cualquier equipo en la red y asegurarse de que no hay colisiones, es decir, nombres duplicados. Se basa en una estructura jerárquica de nombres en forma de árbol donde la raíz es el nodo o dominio raíz (.) Del que derivan todos los demás nodos. Este se divide en otros dominios como, por ejemplo, .com, .edu, .org, .es, etc. A su vez, cada dominio se puede dividir en otros subdominios y así sucesivamente. Las rutas indican empezando por subdominio más abajo en la del árbol hacia el nodo raíz (www.serviciosenred.org).

Los nodos se identifican por un texto (el nombre de dominio) que no se puede repetir en el mismo nivel, pero sí en otros lugares del árbol del espacio de nombres. Lo mismo ocurre con los archivos: no puede haber dos archivos con el mismo nombre en el mismo directorio, pero sí en ubicaciones diferentes. Un dominio es el nodo indicado y todo lo demás del árbol que cuelga de este nodo (piénsese en el ejemplo de un directorio: si se quiere copiar un directorio, se entiende que está formado por el mismo directorio y todos los subdirectorios que contiene). Se entiende por espacio de nombres el conjunto de todos los dominios que forman el árbol DNS.

Dominios de alto nivel

Los siguientes son ejemplos de dominios de alto nivel de algunos países:

  • FR: Francia.
  • AD: Andorra.
  • AQ: Antártida.
  • GB: Gran Bretaña.
  • UK: Reino Unido.
  • DE: Alemania.
  • NL: Países Bajos.
  • AR: Argentina.
  • US: Estados Unidos.

La estructura en árbol (jerárquica) del espacio de nombres proporciona un mecanismo de identificación único de un dominio. No puede existir ningún dominio que tenga exactamente el mismo nombre absoluto FQDN (Full Qualified Domain Name o nombre de dominio completo). Los dominios se leen desde el nodo a la raíz. Así un dominio que corresponda al departamento de administración de la organización example.org dentro del dominio org identifica, por ejemplo, como admin.example.org. Si nos fijamos en el dominio anterior, veremos que termina en punto, es una manera de indicar el dominio raíz. El dominio raíz se define como un dominio sin etiqueta, o mejor con la cadena vacía como etiqueta. Esto provoca que los dominios que se indiquen de manera absoluta terminen con el carácter punto.

El punto final en los nombres de dominio

La mayoría de veces escribimos los dominios como si fueran absolutos, pero son relativos al nodo raíz para que no pongamos el punto final. De nuevo se puede hacer la analogía con las rutas relativas y las rutas absolutas del sistema de archivos.

Un dominio absoluto o FQDN es lo que incluye todos los nodos desde el dominio hasta la raíz (incluida en forma de punto final). Un dominio relativo no incluye la raíz y puede ser relativo al dominio actual. Por ejemplo, dentro del dominio serviciosenred.org del dominio moodle (del aula virtual) es un nombre relativo que hace referencia al nombre absoluto moodle.serviciosenred.org.

Hay que recordar

El sistema de nombres de dominio de Internet DNS utiliza los elementos siguientes:

  • Espacio de nombres. El conjunto de todos los dominios que forman el árbol de dominios.
  • Nombre de dominio. Texto identificativo de un dominio.
  • De absoluto (FQDN). Nombre de dominio absoluto empezando por el nodo y terminando en la raíz (.).
  • Dominio relativo. Nombre de dominio sin calificar.
  • Dominio raíz. Dominio del que derivan todos los demás. Se indica con un punto o con la cadena vacía.

Los nombres de dominio de Internet

En Internet los nombres de dominios siguen una estructura basada en sus inicios, pero que ha ido evolucionando. El nodo raíz se dividió en un conjunto de subdominios denominados TLD (top level domains o dominios de alto nivel). Estos dominios eran como, edu, gov, mil, org, limpio y int. Posteriormente se añadieron otros como cat, name, biz, info, pro, aero, coop y museum. Se querían organizar los dominios para funcionalidad poniendo las empresas en los .com, las organizaciones en los .org, etc. Se vio, sin embargo, la necesidad de poder agrupar los dominios de manera geográfica (ccTLD, country code top level domains) y surgido los famosos identificadores de país. Para cada país se generó un TLD de dos caracteres utilizando el ya existente estándar internacional ISO 3166 (los famosos .es, .fr, etc.).

Caracteres en los nombres de dominio

El estándar DNS indica que los nombres de dominio deben ser de sesenta y cuatro caracteres como máximo, y sólo pueden incluir caracteres latinos, dígitos del 0 al 9 y el guión. Las mayúsculas y minúsculas son indiferentes. Hay mecanismos como el IDNA (año Internationalized domain name o nombres de dominio internacionalizados) para permitir utilizar otros alfabetos en los nombres de dominio.

Origen de los nombres de dominio

Si nos fijamos en los primeros dominios de alto nivel, estaban basados ​​en una visión estadounidense del mundo (de hecho la red ARPANET base de la actual Internet era suya). En extender Internet globalmente y aparecer dominios de alto nivel geográficos, muchas organizaciones se registraron en más de un dominio (por ejemplo, empresa.com y empresa.cat).

Caso práctico: archivo hosts

El archivo hosts es un archivo informático que utiliza un sistema operativo para mapear nombres de ordenadores hacia direcciones IP. Este método es uno de los sistemas para dirigirse por nombre a los ordenadores que hay en una red informática. En algunos sistemas operativos, el archivo de host se utiliza antes de que los otros métodos, tal como el servicio de Domain Name System (DNS). Al contrario que las DNS, el archivo de host es editable por el administrador del ordenador. El archivo hosts es un archivo de texto plano y normalmente se llama hosts.

Donde se encuentra el archivo hosts en Ubuntu y en Windows?

Modifica el archivo hosts en las máquinas virtuales de la siguiente forma:

  • UbuntuServer - ##->dirección IP
  • WindowsServer->dirección IP

De forma que al hacer ping UbuntuServer responda el servidor Linux, WindowsServer el Windows.
