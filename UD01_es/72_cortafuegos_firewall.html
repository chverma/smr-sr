7.2. Cortafuegos (firewall)

El cortafuegos (firewall) es un mecanismo, ya sea de software o hardware, para la protección de redes. Un cortafuegos actúa como una barrera o muro de protección entre dos redes, normalmente un privada y otra pública e insegura como puede ser internet.

La ubicación habitual de un cortafuegos es el punto de conexión de la red interna de la organización con la red exterior que, normalmente, es Internet. Cabe decir que, tal como muestra la imagen inferior, Es frecuente conectar al cortafuegos una tercera red, llamadazona desmilitarizada (DMZ, zona desmilitarizada), En la que se ubican los servidores de la organización que deben permanecer accesibles desde la red exterior.

Por norma general un cortafuegos contiene los siguientes elementos:

  • Filtros: Elementos de hardware o software que se encargan de analizar los paquetes y decidir si aceptarlos o rechazarlos.
  • Nodo bastión: El dispositivo intermediario situado entre las dos redes y que es lo que está expuesto a ataques desde el exterior.

Los objetivos del servicio de cortafuegos son:

  • Garantizar que no se podrá acceder a los recursos internos desde el exterior sin permiso (archivos compartidos, impresoras de red, etc.)
  • Filtrar los paquetes de entrada y salida, permitiendo o denegando el acceso según el origen o el destino, tanto en lo que respecta a las direcciones IP como en los puertos.
  • Utilizar herramientas de software para llevar un control sobre el tráfico de red.

Un cortafuegos se puede configurar de dos formas: permisivo o restrictivo.

En caso de ser permisivo deja entrar, salir y pasar todo el tráfico. Y el administrador se encargará de restringir lo que estime oportuno mediante una lista negra (blacklist). El cortafuegos restrictivo, en cambio, no deja entrar, salir ni pasó nada y es el administrador quien permitirá (abrirá) lo que crea conveniente (whitelist).

Sin embargo, también se puede optar por soluciones mixtas donde algunas direcciones estén restringidas y otras permitidas.

Situación: configurar un cortafuegos restrictivo

El núcleo Linux incluye el subsistema Netfilter, que es usado para manipular o decidir el destino del tráfico de red entre o mediante su red. Todas las soluciones cortafuegos Linux modernas utilizan este sistema para el filtrado de paquetes.

La herramienta de configuración predeterminada del cortafuegos para Ubuntu es ufw. Desarrollado para facilitar la configuración de cortafuegos de iptables, ufw proporciona una forma fácil de usar para crear un firewall basado en host IPv4 o IPv6.

Por defecto el cortafuegos está desactivado, hay activarlo con el comando:

$ sudo ufw enable

Para establecer la configuración predeterminada es necesario usar:

$ sudo ufw default [deny|allow] [incoming|outgoing|routed]

Para abrir un puerto concreto, por ejemplo el 22.

$ sudo ufw allow 22

Para abrir un puerto concreto en un protocolo determinado:

$ sudo ufw allow 22/tcp

Para abrir un puerto en un protocolo determinado y una dirección concreta out hacia el exterior.

$ sudo ufw allow out 22
