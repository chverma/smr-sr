3. El protocolo TCP

El protocolo TCP (Transmission Control Protocol o protocolo de control de transmisión) fue diseñado para funcionar en redes poco fiables. Es un protocolo de la capa de transporte (nivel 4) y el objetivo principal es aportar fiabilidad a las transmisiones basadas en TCP / IP.

La fiabilidad que aporta TCP lo hace ideal para comunicaciones donde queremos asegurar que los datos llegan correctamente, sin pérdidas ni errores. La parte negativa de este fiabilidad es la carga adicional de tráfico que supone en las comunicaciones.

El protocolo TCP proporciona un punto de acceso a las aplicaciones mediante los puertos, números enteros que permiten identificar unívocamente las aplicaciones que están comunicándose.

Los puertos se clasifican de la siguiente manera:

  • Puertos bien conocidos.Los puertos inferiores al 1024 son puertos reservados para el sistema operativo y por protocolos bien conocidos, como HTTP (servidor Web), POP3 / SMTP (servidor de e-mail).
  • Puertos registrados: Los comprendidos entre 1024 y 49151 son denominados "registrados y pueden ser utilizados por cualquier aplicación. Existe una lista pública en la web del IANA donde puede verse qué protocolo usa cada uno de ellos.
  • Puertos dinámicos, privados o efímeros. Los comprendidos entre los números 49152 y 65535. Se asignan de forma dinámica a las aplicaciones clientes al iniciar la conexión.

Socket de internet

Socket designa un concepto abstracto por el cual dos programas (posiblemente situados en ordenadores diferentes) pueden intercambiar cualquier flujo de datos, generalmente de manera fiable y ordenada. Todo socket está definido por una dirección de socket. La dirección de socket es una combinación de tres elementos: una dirección IP, un protocolo de transporte y un número de puerto (por ejemplo: 84.88.125.15, TCP, 2300).
