5. Opciones adicionales del protocolo.

Recuerda que el protocolo DHCP la configuración IP no sólo una dirección IP y la máscara. Para añadir nuevos parámetros se utiliza la palabra reservada options.

option subnet-mask

La opción subnet-mask utiliza para especificar la máscara de subred que se enviará al cliente, si se omite se enviaría la máscara que va asociada con la declaración subnet a la que pertenezca la IP asignada. Lo normal es omitirlo para facilitar el mantenimiento del archivo de configuración. Su sintaxis es la siguiente:

option subnet-mask ip-address;

Ejemplo:

subnet 10.0.0.0 netmask 255.255.255.0 {
   range 10.0.0.10 10.0.0.254;
   option domain-name-servers 8.8.8.8, 8.8.4.4;
   option routers 10.0.0.1;
   option broadcast-address 10.0.0.255;
   option subnet-mask 255.255.255.0;
}

option routers

La opción routers usa para enviarle al cliente su puerta de enlace. Puede especificar una IP numérica o un nombre. Su sintaxis es la siguiente:

option routers ip-address;

ejemplo:

subnet 10.0.0.0 netmask 255.255.255.0 {
    range 10.0.0.10 10.0.0.254;
    option domain-name-servers 8.8.8.8, 8.8.4.4;
    option routers 10.0.0.1;
}


option domain-name-servers

La opción domain-name-servers se emplea para enviar al cliente los servidores DNS que utilizará. Su sintaxis es la siguiente:

option domain-name-servers ip-address [, ip-address ...];

Los servidores se especificarán por orden de preferencia, siendo el primero de la lista el que se utilizará como servidor DNS primario.

Los servidores DNS también se pueden especificar, si queremos, a través de nombres que el servidor DHCP resolverá con la ayuda del servidor de nombres especificado en la configuración de la red del equipo donde se encuentre. La ventaja de usar nombres en lugar de IP, es que si estas cambian, el nombre normalmente no cambiará y por tanto no será necesario modificar la configuración del servidor DHCP, pero el inconveniente es que la respuesta al cliente, que siempre será una IP, tardará un poco más.

En el caso de usar nombres en lugar de IP es aconsejable instalar en el servidor DHCP un servidor DNS, o por lo menos que el servidor DNS esté en el mismo segmento de red que el servidor DHCP, esto agilizará bastante la respuesta los clientes.

Ejemplos:

option domain-name-servers ns.midns.com;
 
option domain-name-servers 8.8.8.8, 8.8.4.4;
subnet 10.0.0.0 netmask 255.255.255.0 {
    range 10.0.0.10 10.0.0.254;
    option domain-name-servers 8.8.8.8, 8.8.4.4;
}
