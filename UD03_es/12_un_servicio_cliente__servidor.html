1.2. Un servicio cliente / servidor

Como suele ocurrir con los servicios cliente / servidor, un equipo puede realizar las dos funciones al mismo tiempo.

El servicio DHCP es uno más de los servicios de red que tienen la estructura cliente / servidor. Los servidores DHCP son los equipos que tienen en ejecución el programa servidor. Es el programa encargado de atender las peticiones de los clientes y ofrecerles la configuración de red, llevando el registro de las IP que concede y el registro de todas las acciones que realiza. Los clientes DHCP son aquellos equipos que realizan peticiones para obtener una configuración de red a un servidor DHCP.

El client DHCP

Un equipo cliente DHCP es un equipo que solicita la IP y otros parámetros de configuración de red a un servidor DHCP en lugar de tenerlos definidos localmente en el equipo.

ISP

ISP: Internet service provider o proveedor de servicio / acceso a Internet. Lo son por ejemplo las empresas Ono, Vodafone, Jazztel, etc.

Si conecta su equipo informático en la red Internet por medio de un ISP (Internet service provider o proveedor de servicio / acceso a Internet), seguramente recibirá una IP dinámica de su proveedor. Cuando se realizaba una llamada telefónica con módem y usando el protocolo PPP (Point-to-Point Protocol, protocolo punto a punto), el proveedor proporcionaba una dirección IP dinámica. Si utiliza ADSL y un router o router, seguramente el router proporciona una dirección IP dinámica privada en el ordenador de casa. Al mismo tiempo, el router obtiene una IP dinámica pública del proveedor. Estas direcciones IP dinámicas son fijas (siempre las mismas) o dinámicas de intervalo (puede ser cualquier IP del conjunto de direcciones IP que tiene disponibles para conceder el servidor DHCP).

IP pública / IP privada

La diferencia entre una IP pública y una IP privada es que la pública es visible para todos los equipos de Internet, mientras que la privada es visible sólo dentro de la misma red local.

El cliente DHCP debe tener en funcionamiento un demonio (daemon) encargado de la gestión de las tareas DHCP por parte del cliente. El programa cliente realiza la parte de negociación encargada al cliente (DHCP discovery, request) y también lleva un registro de las concesiones (leases) recibidas. Este registro es el que utiliza el cliente para volver a pedir la misma IP que tenía anteriormente. Una vez recibida la concesión el programa cliente queda "dormido", pendiente de volver a ejecutar automáticamente cuando sea necesario renegociar la concesión. Sin intervención del usuario, el programa cliente se activa y sigue el procedimiento necesario para renegociar la dirección IP cada vez que el tiempo de la concesión se agota.

configuración cliente

Usualmente las configuraciones clientes se pueden hacer de tres maneras diferentes:

  • archivo de texto: editar directamente los archivos de configuración.
  • menús en modo texto: usando algún programa de menús de interfaz.
  • aplicación gráfica: usando una aplicación de ventanas en el entorno gráfico.

Los programas cliente varían de un sistema operativo a otro y la manera de ejecutarlos también. Generalmente se dispone de un cliente ejecutable en modo texto o órdenes y de una interfaz gráfica (GUI, graphics user interface o interfaz gráfica de usuario) para la configuración. Huelga decir que los sistemas Windows tienden a la configuración gráfica usando ventanas y en la configuración y ejecución interna a escondidas del usuario. Normalmente, en los sistemas GNU / Linux, la configuración se hace usando archivos de texto o como opciones en el orden de ejecución. La interfaz gráfica suele ser un frontal (front-end) para llamar la orden. Según sea el sistema operativo se puede consultar el archivo de registro de las concesiones recibidas por el cliente, el fichero de leases, más o menos detalladamente.

Generalmente el programa cliente se puede configurar para definir su comportamiento en la comunicación con el servidor: información a pedir, información a proporcionar al servidor, opciones por defecto, etc.

El servidor DHCP

El administrador de red es el encargado de pensar la ubicación del servidor o servidores DHCP en la estructura corporativa. Cuanto más complicada sea la topología de la red, más difícil será la gestión. Una red corporativa básica puede disponer de un único servidor DHCP que ofrece sus servicios a todos los equipos de la red. Los clientes pueden estar en una misma subred o en varias subredes, pero todas con conectividad con el servidor DHCP. Este también puede ser el esquema de una red privada en casa, donde un enrutador (el del ISP, por ejemplo) proporciona el servicio DHCP en todos los ordenadores de la casa.

Si la red corporativa crece y pasa a tener subredes segmentadas con cortafuegos las unas de las otras, la configuración del servidor DHCP se complica. Si se quiere continuar con un único servidor para toda la red, será necesario que los cortafuegos (firewalls) dejen pasar los paquetes DHCP entre las subredes y el servidor. Otra opción es poner un servidor DHCP para cada subred o grupos de subredes. Haciéndolo así, la administración de cada servidor es más sencilla pero hay más servidores a administrar. Una red con una casuística completa es la que tiene varios servidores DHCP para diversas partes de la red y cortafuegos entre clientes y servidores que deben permitir el paso de paquetes DHCP.

Si el servidor DHCP es el encargado de dar direcciones IP a los clientes, quienes le proporciona una dirección IP a él? O bien otro servidor DHCP (y podríamos volver a hacer la misma pregunta indefinidamente) o bien el administrador. Usualmente, en una red corporativa el servidor DHCP utiliza una IP estática definida por el administrador. Esto le permite estar siempre disponible para los clientes con la misma IP y no lo hace depender de otro servidor externo.

Hay varios programas servidores DHCP: en modo texto, en modo gráfico, en modo "mágico" (no se ve qué hacen) y en modo Unix (todo basado en ficheros de texto). Cada administrador trabaja con sus herramientas preferidas. Las tareas básicas para aprender a utilizar un servidor DHCP son observar / hacer una lista de la configuración actual, activar / detener el servicio, modificar la configuración, monitorizar los logs (registro de sucesos del servicio) y, evidentemente, saber instalar y desinstalar la aplicación servidor.

Los ficheros de logs (sucesos) recogen los eventos que se quieren monitorizar.

Como la mayoría de servicios de red, el servicio DHCP es un servicio que se ejecuta en segundo plano en forma de demonio. El servidor DHCP siempre está encendido escuchando en el puerto 67 las peticiones que recibe de los clientes. Cuando recibe una petición entrante, el programa ejecutable del servidor DHCP la procesa y pone en marcha todo el mecanismo DHCP pertinente para volver a escuchar nuevas peticiones. De hecho, el servidor siempre escucha peticiones y las procesa simultáneamente (según la configuración).

Los archivos del registro del servicio, donde se anotan las concesiones, permiten mantener la información aunque el servicio se detenga o que el servidor se apague. Al volver a poner en marcha se leerán de nuevo los archivos de registros para saber cuáles son las concesiones que se habían realizado.

Los ficheros de concesiones permiten mantener la coherencia de la asignación de direcciones IP entre paradas del servicio.
