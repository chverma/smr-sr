Pràctica 2: Instal·lació del servei SSH

A. Accés remot mitjançant SSH

Objectiu

L'objectiu d'aquesta activitat és que us habitueu a l'ús del client SSH tant per accedir a equips remots com per transferir fitxers.

SSH permet l'accés segur a recursos remots, ja sigui al sistema de l'equip o a algun dels seus recursos.

Per fer aquesta pràctica haureu de disposar de dues màquines, la local i la remota. En cas que no sigui així, podeu instal·lar el client i el servidor a la mateixa màquina.

Podeu consultar el següent recurs: https://help.ubuntu.com/16.04/serverguide/openssh-server.html

Recursos necessaris

Dos ordinadors, ja que un actuarà de servidor SSH (al que ens connectarem de forma remota), i l'altre de client SSH, des del que ens connectarem. En el seu defecte podem disposar d'una màquina virtual que faça de servidor SSH.

El node que actue de servidor SSH haurà de disposar d'un sistema operatiu basat en Ubuntu.

Activitat

Seguiu les instruccions següents per fer l'activitat:

  1. Instal·leu el programa client i el servidor SSH: openssh-server.
  2. Accediu al terminal del equip remot.
  3. Quina pregunta apareixerà la primera vegada que us connecteu a l'equip remot? Què vol dir?
  4. Llista el contingut de la carpeta a la que has accedit.
  5. Des del client escull un fitxer de l'equip local i copia'l a l'equip remot mitjançant el comandament scp. Explica els paràmetres utilitzats.
  6. Connectat mitjançant el protocol sftp al servidor SSH. Descarrega algun fitxer.

Documenta el procés.

B. Configuració del servei SSH

Objectiu

L'objectiu d'aquesta activitat és que aprengueu a configurar els paràmetres bàsic d'un client i un servidor SSH.

Per fer aquesta pràctica haureu de disposar de dues màquines, la local i la remota. En cas que no sigui així, podeu instal·lar el client i el servidor a la mateixa màquina.

Per a obtenir informació de les opcions de configuració podeu usar:

  • man ssh_config - per al client
  • man sshd_config - per al servidor

Activitats

Seguiu les instruccions següents per fer l'activitat:

  • Configura el client SSH [Linux Mint] de forma que al accedir al servidor s'indique l'usuari per defecte i no siga necessari indicar l'usuari.
  • Configura el servidor de forma que només escolte per l'adreça IP externa i sols puga accedir l'usuari ftp_user. Configura també que el temps de login no podrà superar els 60 segons.

Documenta el procés.

D. Accés remot des de Windows a Ubuntu amb SSH

Objectius

L'objectiu d'aquesta activitat és que aprengueu algunes utilitats per a connectar equips en Windows a servidors SSH basats en Linux tant en mode gràfic com en mode text. 

Activitat. Client SSH per línia de comandament

PuTTY és un client de terminal Transmission Control Protocol, desenvolupat per Simon Tatham, distribuït com a Programari lliure. Inicialment va ser desenvolupat per funcionar sobre el Sistema Operatiu Microsoft Windows però actualment n'existeixen versions per diverses plataformes Unix i Linux, i ja es troben en desenvolupament versions per Mac OS X.

Recursos: Equip amb Windows i equip un sistema basat en Linux amb el servidor OpenSSH instal·lat.

  1. Descarrega't i instal·lat el servidor de finestres Xming X Server (inclou el client SSH putty) en Windows. Més endavant utilitzarem el servidor de finestres.
  2. Connectat a l'equip amb Linux
  3. Fes una captura de la connexió i un altra de la sessió. 
Informació addicional

Windows 10 ja inclou de forma nativa un client i un servidor Open SSH, actualment es troba en la versió beta. Si voleu provar-la (Cómo activar el cliente OpenSSH oculto en Windows 10)

Altres aplicacions les podreu trobar en Free SFTP, SCP and FTP client for Windows

Activitat. Connexió de forma gràfica des de Windows a un equip amb Linux.

Recursos: un equip en Windows i l'altre amb Linux en entorn gràfic. 

  1. Instal·lat en Windows el servidor de finestres xming (si no ho has fet abans) i les fonts xming-fonts
  2. Fes una captura de l'execució d'alguna aplicació gràfica de Linux en Windows.
