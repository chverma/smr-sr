1.2. Diferentes tipos de acceso remoto

Las tareas de acceso y administración remotos se pueden llevar a cabo con medios diferentes, pero siempre estarán condicionados por la red que conecta el equipo que hay que administrar con la estación donde el administrador.

La red impondrá condiciones de:

  • Capacidad : si la conexión no tiene un ancho de banda suficiente no será práctico trabajar con una interfaz gráfica remota. Cuanto más aumente el retraso en la red más frustrante será el trabajo interactivo.
  • Seguridad : es obvio que la comunicación entre el administrador y el equipo remoto no debe ser interceptada por otros usuarios de la red.

Los diferentes medios de administración remota se pueden clasificar en tres categorías básicas:

  • Sesión de trabajo en la consola
  • Sesión de trabajo con interfaz gráfica
  • Cliente o herramienta de administración local

Cada categoría tiene sus puntos fuertes y débiles. Una sesión de trabajo en la consola mediante SSH , o el obsoleto e inseguro Telnet, no exige grandes capacidades en la red. Es el método más ligero e incluso se puede utilizar con comodidad para administrar máquinas muy distantes o con redes de capacidad escasa. Además, es relativamente sencillo automatizar tareas mediante guiones de programación para repetir las mismas operaciones en un conjunto de máquinas.

La administración remota con interfaz gráfica permite abrir a la estación local ventanas de aplicaciones que se ejecutan en el servidor remoto. Incluso permite ver el escritorio completo de la estación remota. Aunque puede ser un método de trabajo cómodo, requiere capacidades de la red que normalmente sólo se encuentran disponibles dentro de una LAN.

Emplear una herramienta de administración localhecha a medida, como un asistente para configurar una impresora remota, o genérica, como un navegador web, intenta conjugar puntos fuertes de las dos técnicas anteriores. En este caso el administrador dialoga con una aplicación local o una página web y no necesita recordar órdenes. Además, dado que la representación gráfica se produce localmente no se exige gran capacidad en la red. El problema de las herramientas específicas es que se trata de un software a instalar en cada estación que el administrador quiera emplear. A menudo esta herramienta sólo está disponible para un sistema operativo o para una versión determinada de este. Este problema se obvia con las interfaces web, ya que un navegador web es un software común que se encuentra en todos los equipos. En la taula.1 puede ver un resumen de las principales ventajas e inconvenientes.

Método de administración ventajas inconvenientes
Trabajo en la línea de comandos

Requisitos mínimos para la red. Es flexible y se puede automatizar.

 Hay que conocer la sintaxis y recordar las órdenes
interfaz gráfica Visual y flexible. No es necesario recordar órdenes. Impone requisitos de capacidad en la red
cliente local Visual y flexible. No es necesario recordar órdenes. No consume muchos recursos en la red. Si no se trata de una interfaz web, es necesario instalar software.

En la actualidad algunas de las herramientas de administración remota más utilizadas son:

  • En modo texto (línea de comandos): telnet, Secure Shell (SSH).
  • En modo gráfico (GUI): VNC, Terminal Server.
  • Cliente local (interfaces web): La mayoría de dispositivos disponen de un servidor web ligero que permite su configuración. También hay aplicaciones como Webmin para servidor basados ​​en Linux.
