3. Acceso remoto con interfaz gráfica

Si la red de comunicaciones tiene un ancho de banda suficiente y un retraso razonable, se pueden utilizar herramientas que permiten trabajar con aplicaciones que se ejecutan en un equipo remoto pero muestran su interfaz gráfica en un terminal local.

Es posible que todo el escritorio visualizado se corresponda con el escritorio de un equipo remoto o combinar en el mismo escritorio aplicaciones locales y remotas. En este último caso, incluso las aplicaciones remotas pueden estar ejecutando en diferentes equipos.

Existen diferentes herramientas y protocolos para llevar a cabo la administración remota mediante una interfaz gráfica. En todos los casos la seguridad debe ser un punto fundamental a considerar, dado que para la misma función de la aplicación estaremos transportando por la red la entrada / salida con el equipo remoto.

Algunas de las posibilidades para llevar a cabo la administración mediante una interfaz gráfica son:

  • VNC
  • FreeNX
  • RDP

VNC (Virtual Network Computing)

VNC es un sistema para ver, y en su caso también controlar, un escritorio remoto. Originalmente fue desarrollado por Olivetti y hay implementaciones gratuitas de las herramientas que permiten trabajar con VNC en sistemas operativos diferentes.

Esta aplicación permite acceder desde un ordenador llamado cliente a otra máquina llamada servidor, separados físicamente pero, sin embargo, conectados de forma. Una vez se ha establecido la comunicación, el equipo cliente puede utilizar el equipo servidor sin limitación, salvo las impuestas por el ancho de banda de la red. El resultado es una visualización en pantalla del equipo remoto, del escritorio y de todos sus programas. Para lograr esto, es necesario instalar dos programas: un servidor VNC en la máquina a la que queramos acceder y un visualizador VNC en la máquina cliente. Si utilizamos un visualizador en un sistema operativo dado y un servidor en otro distinto, ejecutaremos todo el sistema servidor en el cliente. De este modo, es posible ejecutar un sistema Windows completo desde un Macintosh o GNU / Linux,o hacer cualquier combinación que podamos imaginar. Como se puede ver en la figura 34, se puede ejecutar un sistema GNU / Linux desde un equipo con el sistema operativo MacOS o Windows.

Funcionamiento del protocolo VNC

El protocolo de red que se utiliza, RFB (r emote framebuffer protocolo ), también es libre. Aunque las funciones iniciales del protocolo RFB se limitaban a transmitir los acontecimientos desencadenados por el usuario en el teclado y el ratón locales en la estación remota, ya enviar en dirección contraria el contenido gráfico de la pantalla, las últimas versiones han sido ampliadas para soportar más funcionalidades, como por ejemplo la transferencia de archivos.

La memoria de imagen ( framebuffer ) es el área de memoria que almacena los datos que definen la imagen gráfica que aparece en la pantalla. Normalmente todas las pantallas actuales están basadas en píxeles, es decir, muestran mapas de bits en vez de imágenes vectoriales.

La memoria de imagen contiene la información necesaria para definir el estado de cada píxel de la pantalla. La cantidad de información necesaria dependerá de la resolución y la profundidad de color (la cantidad de bits necesarios para codificar el color, y quizás la transparencia, de cada píxel).

Dado que VNC está basado en una memoria de imagen, puede trabajar con cualquier sistema de ventanas, ya sea X Window, Windows u otro. En este sistema siempre llama cliente la estación en que se encuentra el usuario (equipo local) y se denomina servidor del equipo que define el contenido de la pantalla (equipo remoto). Esta nomenclatura es inversa a la que utiliza el sistema X Window.

Todo el sistema está pensado para trabajar con clientes ligeros de manera que el cliente pueda implementar sin muchos recursos. De hecho todo el protocolo y el funcionamiento del sistema intentan ser muy sencillos. El pilar fundamental son las actualizaciones gráficas que el servidor envía hacia el cliente, en forma de rectángulos, para modificar los datos de la memoria de imagen. Estos mensajes pueden emplear diferentes codificaciones, pero hay unas básicas que todos los clientes y servidores VNC deben conocer.

Una característica fundamental del sistema VNC es que el cliente no conserva ningún estado. Así es posible cerrar un cliente y volverlo a abrir, incluso en otro equipo, como si se tratara de un monitor virtual.

En los sistemas GNU / Linux es posible exportar mediante VNC el escritorio actual o incluso un terminal de texto, o bien crear un escritorio virtual nuevo. Los sistemas Windows no permiten la creación de escritorios virtuales nuevos, sólo permiten exportar en red el único escritorio disponible. En el momento de exportar un escritorio se pueden definir dos contraseñas diferentes, una para aquellos usuarios que podrán controlar de forma remota este escritorio y otra para aquellos usuarios que sólo lo podrán ver, pero no interactuar con el teclado y el ratón .

El proceso para utilizar VNC es muy simple:

  1. En la estación remota hay que ejecutar un servidor VNC para exportar el escritorio de trabajo o uno nuevo.
  2. En la estación local se utilizará un cliente VNC para interactuar con el servidor remoto. Obsérvese que RFB no es un protocolo seguro. Al usar VNC en una red que no sea de confianza, habrá que combinar su uso con SSH u otra tecnología para crear una VPN.

La mayor parte de servidores VNC incluye un pequeño servidor web que ofrece la descarga de un applet Java que se puede usar como cliente VNC. De este modo, en el ordenador local hace falta un navegador con soporte Java para poder acceder al escritorio remoto.

GPL: La licencia pública general (GPL, del inglés general public license) es un tipo de licencia para software que permite la copia, distribución (comercial o no) y modificación del código, siempre que cualquier modificación se continúe distribuyendo con la misma licencia GPL. La licencia GPL no permite la distribución de programas ejecutables sin el código fuente correspondiente ni con una oferta de cómo obtenerlo gratuitamente.

VNC, muchas herramientas y diferentes implementaciones

Si el desarrollo original de VNC tiene algún heredero oficial, este es RealVNC. Un producto desarrollado por la compañía del mismo nombre que está formada por algunos de los desarrolladores originales. Las herramientas de RealVNC se ofrecen con licencias diferentes: la edición Free, bajo licencia GPL y el resto, con licencias privativas.

Otra implementación bastante extendida es TightVNC, software libre con licencia GPL que, aunque es plenamente compatible con las herramientas originales, introduce algunas extensiones al protocolo RFB. Estas extensiones sólo se podrán emplear en utilizar TightVNC, tanto en cliente como en el servidor. Entre las extensiones introducidas por TightVNC hay codificaciones nuevas que reducen la carga de la red y la posibilidad de transmitir archivos.

RDP (Remote Desktop Protocol)

Remote Desktop protocolarias l (RDP) es un protocolo propietario desarrollado por Microsoft que permite la comunicación en la ejecución de una aplicación entre un terminal (mostrando la información procesada que recibe del servidor) y un servidor Windows (recibiendo la información dada por el usuario en el terminal mediante el ratón o el teclado).

El modo de funcionamiento del protocolo es sencillo. La información gráfica que genera el servidor es convertida a un formato propio RDP y enviada a través de la red al terminal, que interpretará la información contenida en el paquete del protocolo para reconstruir la imagen a mostrar en la pantalla del terminal. En cuanto a la introducción de órdenes en el terminal por parte del usuario, las teclas que pulse el usuario en el teclado del terminal así como los movimientos y pulsaciones de ratón son redirigidos al servidor, permitiendo el protocolo un cifrado de los mismos por motivos de seguridad. El protocolo también permite que toda la información que intercambien cliente y servidor sea comprimida para un mejor rendimiento en las redes menos veloces.

Este servicio utiliza por defecto el puerto TCP 3389 en el servidor para recibir las peticiones.1 Una vez iniciada la sesión desde un punto remoto del ordenador servidor mostrará la pantalla de bienvenida de Windows, no se verá lo que el usuario está realizando de forma remota.

Este servicio tiene diferentes tipos de aplicaciones: se utiliza frecuentemente para el acceso remoto a la administración de equipos, pero también es cada vez más utilizado en la gestión de servicios de terminal o clientes ligeros ( thin clients ).

características

  • Permite el uso de colores de 8, 16, 24 y 32 bits.
  • Cifrado de 128 bits utilizando el algoritmo criptográfico RC4. Los clientes más antiguos pueden utilizar cifrados más débiles.
  • Permite seguridad a nivel de transporte Transport Layer Security (TLS).
  • El redireccionamiento del audio permite al usuario ejecutar un programa de audio en una ventana remota y escuchar el sonido en el ordenador local.
  • El redireccionamiento del sistema de archivos permite a los usuarios utilizar sus archivos locales en una ventana remota.
  • Permite al usuario utilizar la impresora local al estar conectado al sistema remoto.
  • El redireccionamiento de puertos permite utilizar los puertos serie y paralelo directamente.
  • El portapapeles puede compartirse entre los ordenadores local y remoto.
  • Programas remotos: Aplicaciones con archivos del lado del cliente.
  • Las aplicaciones ( seamless windows ) remotas pueden funcionar en una máquina cliente servida por una conexión remota.
  • Soporte de varios monitores. La sesión puede mostrarse en dos monitores.

Team Viewer

TeamViewer es un software que su función es conectarse remotamente a otro equipo. Entre sus funciones están: compartir y controlar escritorios, reuniones en línea, videoconferencias y transferencias de archivos entre ordenadores.

características

TeamViewer está disponible para los sistemas operativos Microsoft Windows, Mac OS X, Linux, iOS, Android, Windows Phone 8, Windows 10 Mobile, BlackBerry y Windows RT. También es posible el acceso a un equipo remoto mediante un navegador web. Aunque el principal objetivo de la aplicación es el control remoto, también incluye funciones de trabajo en equipo y presentación.

TeamViewer puede ser utilizado por los usuarios no comerciales sin ningún tipo de coste, sin embargo dispone de versiones Bussines, Premium y Corporate.

funcionamiento

El software se puede instalar con un procedimiento de instalación ordinario, aunque la versión "Quick Support" ejecutará sin necesidad de instalación. Para conectarse a otro equipo, hay que TeamViewer se ejecute en ambas máquinas.

La instalación de TeamViewer, requiere acceso de administrador, pero una vez instalado puede ser ejecutado por cualquier usuario.

Cuando el software se inicia a un ordenador, genera un ID y contraseña (el usuario puede definir una propia si lo desea). Para establecer una conexión entre un equipo local y uno remoto, el usuario del equipo local se pondrá en contacto con el otro e introducir el ID y contraseña de éste, mientras que el equipo remoto requiere el ID y contraseña del local, es decir, a la inversa.

Para empezar una reunión en línea, el moderador proporciona el ID de la reunión los participantes, los cuales se añaden al encuentro a partir de la versión completa de TeamViewer o la versión para navegador mediante el ID mencionada. También es posible programar una reunión anticipadamente.

seguridad

Las sesiones de TeamViewer se codifican mediante el algoritmo RSA (1024 bits) de cifrado de clave pública, y AES (256 bits).

En la configuración predeterminada, el software utiliza uno de los servidores de TeamViewer.com para conectar el equipo local y el remoto. No obstante, en el 70% de los casos después del de acceder al protocolo de enlace, se establece una conexión directa vía UDP o TCP. El resto de conexiones se enrutan a través del router de la red de TeamViewer GmbH (Tunnel TCP o HTTP).

Escritorio remoto de Chrome

Chrome Remote Desktop permite a los usuarios acceder de forma remota otro ordenador a través del navegador Chrome o Chromebook. Los ordenadores pueden estar disponibles sobre una base ad hoc a corto plazo para escenarios como el apoyo a distancia, o sobre una base más a largo plazo para el acceso remoto a las aplicaciones y archivos. Todas las conexiones están totalmente asegurados.

Chrome Remote Desktop es totalmente multiplataforma. Proporcionar asistencia a los usuarios de Windows, Mac y Linux remotas, o acceder a su Windows (XP y superior) y Mac (OS 10.6 y 10 por encima de) los escritorios en cualquier momento, todo ello desde el navegador Chrome en prácticamente cualquier dispositivo, incluyendo los Chromebook.
