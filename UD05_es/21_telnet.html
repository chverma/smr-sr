2.1. Telnet

telnet es un protocolo que emula un terminal remoto para conectarse a una máquina multiusuario. Al igual que con el protocolo FTP, es necesario el software necesario y un protocolo específico para este servicio.

El puerto que se utiliza es generalmente el 23. Sólo sirve para acceder en modo terminal, es decir, sin gráficos, pero fue una herramienta muy útil para arreglar problemas a distancia. También se ha utilizado para consultar datos a distancia, como datos personales accesibles a través de la red, información bibliográfica, etc. Los requisitos para acceder a este servicio son muy sencillos: conocer el nombre y la dirección del servidor remoto y estar habilitado para poder utilizarlo mediante un identificador de usuario y una contraseña. Normalmente se llama a una aplicación externa que es la que realiza la conexión. En este caso se indicará la máquina y el login: telnet://maquina.dominio@login.

Su principal problema es de seguridad, ya que todos los nombres de usuario y contraseñas necesarias para entrar en las máquinas viajaban por la red sin cifrar. Esto hacía que cualquier persona que espías el tráfico de la red pudiera obtener estos datos y de esta manera acceder también a las máquinas. Se dejó de utilizar casi totalmente, hace unos años, cuando apareció y se popularizó el SSH, que se puede describir como una versión cifrada de Telnet.
